# Idea Factory

## Overview

**Idea Factory** is a guided concept-development system for independent thinkers working on product, framework, or policy concepts.

Its purpose is to help a user:

1. understand an idea clearly,
2. evaluate whether it is worth further effort,
3. structure it into a small concept artifact,
4. and share it when that is warranted.

The system is not a generic chatbot, a free-form CMS, or a site builder with AI attached. It is a governed process for turning rough ideas into clear, evaluated, and optionally shareable concept sites.

## Core value order

Idea Factory is built around a deliberate order of value:

1. **Understand the idea**
2. **Evaluate the idea**
3. **Structure the idea**
4. **Compile a concept artifact**
5. **Share it if warranted**

That order matters. The system should not rush toward polish or publication before the concept is clear and honestly assessed.

## Who it is for

Phase I is aimed at **independent thinkers** developing:

- product concepts,
- framework concepts,
- policy concepts.

The user owns the concept and all important decisions. The LLM guides the process, challenges weak thinking, and helps maintain structure and evaluation integrity.

## What it produces

The main output is a **small concept site** or **idea brief with pages**.

A standard artifact typically includes:

- an overview,
- the problem,
- the core idea,
- how it works,
- who it is for,
- an evaluation page,
- and open questions.

The artifact is meant to be readable, structured, and honest. It should help the user think, decide, and communicate more clearly than a single large document would.

## How the system behaves

Idea Factory uses a stable mentor-like operating style:

- calm,
- restrained,
- deliberate,
- explicit about uncertainty,
- willing to disagree,
- unwilling to flatter the user into false confidence.

The LLM is not there to celebrate every idea. It is there to guide the user through a disciplined process.

## Why this approach is different

The system is designed around a few strong constraints:

- the user owns the concept,
- the LLM may guide but not silently decide,
- evaluation is part of the artifact,
- protected judgments cannot be manually overwritten,
- progress must be visible,
- stopping points must be explicit,
- endless sessions should be avoided.

This makes the system closer to a concept compiler than to a note-taking tool.

## Phase I at a glance

Phase I includes:

- a six-stage workflow,
- an explicit evaluation framework,
- a governed editability model,
- linked projects instead of branches,
- a completion model,
- and a progress efficiency model that helps control session length and reduce waste.

The goal of Phase I is not to solve technical design or implementation planning. It is to help a user move from a rough idea to a clear, evaluated concept artifact with an honest next step.
