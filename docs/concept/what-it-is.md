# What Idea Factory Is

## Working definition

**Idea Factory** is a guided concept-development system that helps a user turn a rough idea into a clear, evaluated, and optionally shareable concept artifact.

It is designed to help the user move in a deliberate order:

1. understand the idea,
2. evaluate the idea honestly,
3. structure it into a coherent concept model,
4. compile that model into a small concept site or idea brief,
5. and share it only when that is warranted.

This order is intentional. The system should not rush toward polish or publication before the idea is clear enough to deserve it.

## What kind of system it is

Idea Factory is best understood as a **guided concept system**.

It combines several roles that are often separated in other tools:

- a thinking aid,
- a structuring aid,
- an evaluation aid,
- a concept compiler,
- and a controlled publication layer.

The user brings the concept. The system helps the user clarify it, test it, organize it, and express it.

## What it is not

Idea Factory is **not** primarily any of the following:

### Not a generic chatbot

A chatbot can help discuss an idea, but it often does not preserve structure, process state, evaluation history, or clear stopping conditions.

Idea Factory is meant to guide a concept through defined stages and preserve what happened along the way.

### Not a free-form CMS

The system does not treat the artifact as an unrestricted document that the user can rewrite however they want.

Phase I uses **prompt-based editing**, protected evaluative layers, and governed changes. This preserves the trustworthiness of the resulting concept artifact.

### Not a note-taking tool

The purpose is not to collect notes indefinitely.

The purpose is to turn an idea into a clearer and more decision-ready artifact.

### Not a site builder with AI attached

Although the output may be a concept site, the value of the system is not mainly visual or presentational.

The value is in the process:

- clarification,
- evaluation,
- structure,
- governance,
- and controlled publication.

### Not a technical design tool by default

Technical design may come later, but it is not the center of Phase I.

Phase I is concerned with concept quality, concept structure, and concept readiness, not implementation architecture.

## The central object

The central object in Idea Factory is the **project**.

A project represents one coherent idea effort. It contains:

- the concept in its current form,
- the evolving concept artifact,
- the process history,
- the evaluation record,
- and the current recommendation or next action.

If a materially different idea emerges, the system should not silently blend it into the same project. Instead, it should recommend creating a **linked project** and preserving the lineage between them.

## The core output

The main output of Phase I is a **small concept site** or **idea brief with pages**.

That artifact is meant to help the user do three things:

1. think more clearly,
2. decide more honestly,
3. explain the idea more effectively.

A standard artifact usually contains:

- an overview,
- the problem,
- the core idea,
- how it works,
- who it is for,
- an evaluation page,
- and open questions.

The artifact is therefore both:

- **explanatory**, because it helps describe the concept,
- and **evaluative**, because it shows the current judgment of the concept.

## The role of the user and the LLM

Idea Factory is built around a deliberate role split.

### The user owns

- the concept direction,
- the title,
- the final strategic decisions,
- whether to continue, pause, stop, or publish,
- and challenges to the system's evaluation.

### The LLM owns

- process guidance,
- stage discipline,
- evaluation integrity,
- recommendation output,
- convergence judgments,
- and intervention when the session becomes inefficient, abusive, or harmful.

This balance is central. The system should be opinionated, but it should not silently take control of the concept.

## Why the system exists

Many ideas do not fail because they are impossible. They fail earlier because they remain:

- vague,
- scattered,
- over-polished before being tested,
- or trapped in large unreadable documents.

Idea Factory exists to create a better path between:

- a rough thought,
- a structured concept,
- a serious evaluation,
- and a shareable artifact.

## The governing philosophy

The system is built on a few strong principles:

- the user is in charge of the concept,
- the process must stay finite,
- the system must be honest about what it knows and does not know,
- the artifact should not imply confidence that has not been earned,
- and stopping, pausing, or reframing are valid outcomes.

This means the system is not trying to maximize interaction. It is trying to maximize the quality and usefulness of the progress made.

## Phase I summary

In Phase I, Idea Factory is a governed concept-development system that helps an independent thinker move from a rough idea to:

- a clarified concept,
- a structured artifact,
- an explicit evaluation,
- and a clear next step.

That next step may be:

- continue,
- revise,
- create a linked project,
- pause,
- stop,
- or prepare the artifact for sharing.

That is what Idea Factory is.
