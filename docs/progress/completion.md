# Completion Model

Idea Factory should not feel endless. A project needs clear stopping points, clear handoff points, and a clear definition of what "done for now" means.

The completion model exists to answer:

- what is complete
- what is complete enough for now
- what is paused
- what is stopped
- what is publishable
- what still belongs to a later phase

## Completion exists at three levels

### Step completion

A single unit of work is complete.

Examples:

- the audience has been defined
- the first structure has been approved
- the evaluation has been completed

### Session completion

A working session is complete for now.

A session is complete when:

- a meaningful milestone has been reached
- the next useful step is clear, but should happen later
- further interaction now would produce diminishing returns
- the system has reached a justified pause or stop recommendation

### Project completion

A project is complete enough for its current goal.

This is not the same as perfection. It means the project is sufficiently developed for the user's present purpose.

## Goal-relative completion

Completion should always be evaluated relative to the project's current goal.

Examples of goals:

- clarify the idea
- evaluate whether it is worth pursuing
- create a concept artifact
- prepare for trusted sharing
- prepare for public sharing

A project may be complete enough for one goal while still incomplete for another.

## Core end states

### In progress

The project is still being actively developed.

### Complete for current stage

The current workflow stage is done and the project can move forward.

### Complete enough for now

This is one of the most important states in Idea Factory.

It means:

- the current goal has been met
- the major uncertainties are visible
- the remaining work is known
- continuing immediately would have low marginal value

This should be treated as a valid success state, not as a compromise.

### Paused

The project is not being actively advanced right now.

Reasons may include:

- insufficient value at present
- waiting for more evidence
- limited budget or attention
- the user wanting to step away and return later

### Stopped

The idea should not continue in its current form.

Reasons may include:

- too weak
- too derivative
- too unclear
- not worth more effort
- disallowed or harmful purpose

A stopped project is not a failed process. It is often a successful decision outcome.

### Reframed into a new linked project

The strongest direction that emerged no longer belongs inside the current project.

### Publishable

The concept artifact is fit for the chosen audience.

### Published

A specific version has been intentionally exposed and preserved.

## What makes a project complete enough for Phase I

A Phase I project is complete enough when all of the following are true:

1. the concept is clearly defined
2. the concept has been structured into a coherent artifact
3. the concept has been honestly evaluated
4. the next action is explicit
5. the current artifact matches the current goal

For example, a project may be complete enough for trusted sharing even though it has not yet reached technical design.

## What Phase I does not require

Phase I does not require:

- implementation design
- technical architecture
- infrastructure decisions
- full market validation
- legal or business planning
- perfect wording on every page

This boundary is important. It gives the process a real finish line.

## Session closeout

Every session should end with an explicit closeout that states:

- what changed
- what is now complete
- what remains
- current project state
- current recommendation
- the next step

This is how the system gives the user a sense of closure without pretending the project is fully final.

## Why this model matters

Without a completion model, the system risks becoming:

- open-ended
- expensive
- frustrating
- unclear about stopping

With a strong completion model, the system can say clearly:

- this is enough for now
- this should pause
- this should stop
- this is ready to share
- this should become a new linked project
